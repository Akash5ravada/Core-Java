JDK, JRE, and JVM

1. JVM (Java Virtual Machine)

->JVM is a virtual machine that executes Java bytecode and makes Java platform-independent.

    Loads Java class files.
    Verifies bytecode.
    Executes bytecode.
    Manages memory.
->Functions of JVM
    Performs Garbage Collection(Garbage Collection (GC) is an automatic memory management process in Java that removes unused objects from memory (Heap) to free up space.).


->Flow

        .java file
            ↓
        Compiler (javac)
            ↓
        .class file (Bytecode)
            ↓
        JVM
            ↓
        Machine Code
            ↓
        Output

Key Point: JVM is responsible for running Java programs.


2. JRE (Java Runtime Environment)

->JRE provides the environment required to run Java applications.

->JRE Contains
    JVM
    Core Java Libraries
    Supporting files
->Purpose
    Runs Java programs.
    Does NOT contain development tools like the compiler.
->Diagram
    JRE
     ├── JVM
     └── Java Libraries

Key Point: If you only want to run Java applications, JRE is sufficient.


3. JDK (Java Development Kit)

->JDK is a complete package used for developing, compiling, and running Java applications.

->JDK Contains
    JRE
    JVM
    Development Tools
->Important Tools
    Tool	    Purpose
    javac	    Compiles Java code
    java	    Runs Java programs
    javadoc	    Generates documentation
    jar	        Creates JAR files
    jdb	        Java debugger
->Diagram
    JDK
     ├── JRE
     │    ├── JVM
     │    └── Libraries
     └── Development Tools

Key Point: Developers install JDK because it contains everything needed to develop and run Java programs.


4.Bytecode in Java

->Bytecode is the intermediate code generated by the Java compiler (javac) after compiling a Java source file (.java).
It is stored in a .class file and is executed by the JVM (Java Virtual Machine).

->Flow of Bytecode
    Hello.java
        ↓
    javac (Compiler)
        ↓
    Hello.class (Bytecode)
        ↓
    JVM
        ↓
    Machine Code
        ↓
    Output

->Example
    Source Code (Hello.java)
    public class Hello {
        public static void main(String[] args) {
            System.out.println("Hello Java");
       }
    }

->Compile
    javac Hello.java

After compilation, a file named Hello.class is created. This .class file contains bytecode.

->Run
    java Hello

The JVM reads the bytecode and converts it into machine code for the operating system.

->Why Bytecode is Important
    Platform Independence
    The same .class file can run on Windows, Linux, and macOS.
    Only the JVM needs to be different for each platform.
    Security
    JVM verifies bytecode before execution.
    Performance
    JVM uses the JIT (Just-In-Time) Compiler to convert frequently used bytecode into native machine code.


5. Compiler -> A Compiler translates the entire source code into machine code (or bytecode) at once before execution.

6. Interpreter -> An Interpreter translates and executes code line by line at runtime.
